## 🔍 關鍵技術發現

### 1. BSR 網站回傳格式變更 (重大)
```
舊: POST → HTML 內嵌 <table class='table_blue'> 彙總資料
新: POST → 表單頁 + 「下載 2330 CSV」連結
         → GET bsContent.aspx?StkNo=2330&RecCount=62 → CSV (big5, ~188KB)
         → 逐筆明細 (3085行, 434券商) → 需自行加總
```
**解決**: `_parse_result` 改為先檢查 CSV 連結 → 下載 CSV → 解析彙總 → fallback 舊格式

### 2. TWSE API Rate Limiting
- 連續大量請求後 TWSE 回傳非 JSON (空值)
- 非永久性錯誤，重試 2-3 次即可恢復
- **解決**: `stock_daily_spider.fetch_daily()` 加入 max 3 retry, exponential backoff

### 3. EOD Pipeline Module Path
- `from analytics.risk_assessor` vs `from src.analytics.risk_assessor` 在 sys.modules 不同
- 需統一 path 指向同一模組物件

### 4. S 評級邊界條件
- premium=2% / risk=10% 精確等於門檻值 → 正確退化為 A (不是 S)

## 🏗️ 最終系統架構

```
BSR 網站 ──→ ddddocr OCR ──→ BsrClient (session/captcha/CSV)
                                      ↓
                              BrokerBreakdownSpider
                                      ↓ collect_only → validate → flush
                              ┌───────────────────┐
                              │  broker_breakdown  │ ← PostgreSQL
                              └────────┬──────────┘
                                       ↓
                              ChipProfiler.analyze()
                              (黑名單比對 10 筆)
                                       ↓
                              {symbol: risk_ratio}
                                       ↓
                    ┌───────────────────┴───────────────────┐
                    │                                        │
            PremiumCalculator                      RiskAssessor
            (轉換價值+溢價率)                      (S/A/B/C 評級)
                    │                                        │
                    └────────── assess(premium, risk) ────────┘
                                        ↓
                              daily_analysis_results
                              (final_rating, broker_risk_pct)
                              trading_signals (BUY/HOLD/AVOID)
```
