# Phase 4: 批次處理遷移

## 概述

Phase 4 將批次處理邏輯遷移至 Feapder 框架，實現全市場歷史資料補檔 (Backfill) 功能。

### 核心功能

1. **BatchSpider**: 全市場歷史資料批次抓取
2. **Checkpoint Manager**: 斷點續傳機制
3. **並發控制**: 支援多線程/多程序抓取

### 任務邊界

**Phase 4 負責範圍**:
- 全市場歷史資料補檔 (過去 5 年)
- 斷點續傳 (Checkpoint)
- 批次排程整合

**Phase 5 負責範圍**:
- Agent 協調層
- 排程監控
- 告警整合
