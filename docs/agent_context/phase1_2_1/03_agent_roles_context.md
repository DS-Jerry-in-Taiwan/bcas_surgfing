# Phase 1.2.1 - Agent 角色職責

## 🏗️ @INFRA
- **職責**: 檢查網路連線（此階段由 Code 處理，INFRA 輔助）。
- **重點**: 無特殊變更。

## 📐 @ARCH
- **職責**: 定義「容錯標準」。
- **重點**: 
    - 決定系統在「資料不完整」時的行為模式。在本專案中，允許僅使用 TWSE 資料進行開發驗證。

## 💻 @CODER
- **職責**: 實作 `try-except` 與 Logging 機制。
- **重點**:
    - 捕捉 `socket.gaierror` (DNS 錯誤)。
    - 設定合理的 Timeout (例如 5秒)，避免 Pipeline 卡死在等待回應。

## 🧪 @ANALYST
- **職責**: 驗證「降級執行」的正確性。
- **重點**:
    - 確保系統不會因為 `cb_list` 為空而在後續步驟（如隨機抽樣）拋出 `IndexError`。
