# Phase 1.2 - Agent 角色職責

## 🏗️ @INFRA
- **職責**: 調整資料夾結構以區分 `Master Data` (靜態清單) 與 `Daily Data` (動態行情)。
- **重點**: 確保 `data/raw/master/` 路徑存在。

## 📐 @ARCH
- **職責**: 定義「全市場清單」的獲取策略。
- **重點**:
    - 台股上市與上櫃通常在不同 API，需設計聚合 (Aggregation) 邏輯。
    - CB 清單需確認包含「對應現股代碼」欄位，以便後續 Mapping。

## 💻 @CODER
- **職責**: 實作兩階段爬蟲 (List -> Detail)。
- **重點**:
    - 優先攻克 `StockMasterFetcher` 與 `CBMasterFetcher`。
    - 確保爬蟲能自動過濾已下市或暫停交易的標的（如果 API 有提供狀態）。

## 🧪 @ANALYST
- **職責**: 把關清單的「覆蓋率」與日資料的「可用性」。
- **重點**:
    - 驗證流程必須是：先跑清單 -> 再挑標的 -> 再跑日資料 -> 最後檢查檔案。
    - 嚴格檢查 CB 資料是否包含關鍵的「交易量」與「價格」。
